# H100/GB10 Result Differentiation Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Route `harm_dimensions_v2` evaluation results to hardware-specific subdirectories (`H100/` or `GB10/`) via a `--gpu` flag, and migrate existing GB10 results into place.

**Architecture:** Add a required `--gpu <H100|GB10>` argument to each run script; the flag sets `OUTPUT_DIR` and selects the correct config file automatically. Existing tracked GB10 results are moved via `git mv`. `.gitignore` is updated to un-ignore both subdirs.

**Tech Stack:** Bash, git

### Task 3: Add `--gpu` flag to `run_harm_v2_sequential.sh`

**Files:**
- Modify: `scripts/run_harm_v2_sequential.sh`

**Step 1: Replace the variable declarations block**

Find these lines (after `SCRIPT_DIR` / `REPO_ROOT`):
```bash
PYTHON="$(which python)"
OUTPUT_DIR="$REPO_ROOT/data/results/vllm/harm_dimensions_v2"
CONFIG="$REPO_ROOT/config/vllm_jury_config.yaml"
```

Replace with:
```bash
PYTHON="$(which python)"
GPU_LABEL=""

# Parse arguments
while [[ $# -gt 0 ]]; do
    case $1 in
        --gpu)
            GPU_LABEL="$2"
            shift 2
            ;;
        *)
            echo "Unknown option: $1"
            echo "Usage: $0 --gpu <H100|GB10>"
            exit 1
            ;;
    esac
done

if [[ -z "$GPU_LABEL" ]]; then
    echo "ERROR: --gpu flag is required. Use --gpu H100 or --gpu GB10"
    exit 1
fi

if [[ "$GPU_LABEL" == "GB10" ]]; then
    CONFIG="$REPO_ROOT/config/vllm_jury_config_gb10.yaml"
else
    CONFIG="$REPO_ROOT/config/vllm_jury_config.yaml"
fi

OUTPUT_DIR="$REPO_ROOT/data/results/vllm/harm_dimensions_v2/$GPU_LABEL"
```

**Step 2: Update the log header to show GPU label**

Find:
```bash
log "harm_dimensions_v2 — sequential evaluation"
```
Replace with:
```bash
log "harm_dimensions_v2 — sequential evaluation [$GPU_LABEL]"
```

**Step 3: Verify script exits with error when `--gpu` is missing**
