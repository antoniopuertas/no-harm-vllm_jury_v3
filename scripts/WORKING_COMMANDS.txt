========================================
WORKING COMMANDS (Copy & Paste)
========================================

STEP 1: STOP CURRENT HUNG EVALUATION
-------------------------------------
bash scripts/STOP_AND_FIX.sh


STEP 2: RUN WITH WORKING CONFIG
---------------------------------

OPTION A - Single GPU (RECOMMENDED - Most Reliable)
====================================================
nohup python scripts/run_full_vllm_evaluation.py --dataset medmcqa --instances 1000 > logs/medmcqa_single_$(date +%Y%m%d_%H%M%S).log 2>&1 &

Expected time: ~50-60 minutes for 1000 samples


OPTION B - Hybrid GPU (OLMo uses 2 GPUs, others single)
========================================================
nohup python scripts/run_full_vllm_evaluation.py --dataset medmcqa --instances 1000 --config config/vllm_jury_config_hybrid_gpu.yaml > logs/medmcqa_hybrid_$(date +%Y%m%d_%H%M%S).log 2>&1 &

Expected time: ~40-45 minutes (slightly faster due to OLMo speedup)


MONITOR PROGRESS
================
tail -f logs/medmcqa_*.log


CHECK IF RUNNING
================
ps aux | grep run_full_vllm_evaluation


WATCH GPU USAGE
===============
watch -n 1 nvidia-smi


WHY DUAL-GPU DIDN'T WORK
=========================
Gemma3-27B has a known issue with tensor parallelism in vLLM v0.15.1
It hangs during NCCL initialization when tensor_parallel_size=2
This is a vLLM/model compatibility issue, not your setup


BEST APPROACH
=============
Use Single GPU with 1000 instances:
- Works reliably
- Completes in ~1 hour
- Much faster than 6150 instances (5-6 hours)
- No tensor parallelism issues
